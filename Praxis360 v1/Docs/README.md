# Praxis360 Documentation

Welcome to the official Praxis360 documentation.

This folder contains the complete functional, technical and design documentation of the project.

The documentation is considered part of the product.

Every Sprint must keep these documents up to date.

---

# Documentation Structure

## Blueprint.md

Defines the functional vision of Praxis360.

Read this document first to understand the product.

---

## Architecture.md

Describes the software architecture, project structure and development rules.

---

## ProductBook.md

Describes every functional module of the application.

Explains how features should behave.

---

## DesignBible.md

Defines the complete design system.

Includes colors, typography, spacing, reusable components and UI principles.

---

## MotionGuide.md

Defines every animation and interaction.

Explains how the application should feel.

---

## SprintBook.md

Records the complete development history.

Tracks completed, current and future Sprints.

---

## Roadmap.md

Defines the long-term evolution of Praxis360.

---

## Assets/

Contains documentation assets.

Folders include:

- Branding
- Components
- Flows
- Logos
- Screens
- Wireframes

Praxis360 is developed as a long-term product.

Consistency, maintainability and quality always take priority over development speed.
