# Praxis360 Documentation

Welcome to the official Praxis360 documentation.

This documentation is the single source of truth for the project.

It defines the product vision, business rules, architecture, design system and development process.

Documentation is considered part of the product and evolves together with the code.

Every Sprint must keep these documents up to date.

---

# Documentation Structure

## Blueprint.md

Defines the overall development framework of Praxis360.

It contains the project's core principles, development rules and functional boundaries.

Read this document first.

---

## ProductVision.md

Defines the business vision of Praxis360.

Explains the product promise, customer experience, UX philosophy and business principles.

Every feature, screen and business model must comply with this document.

---

## Architecture.md

Describes the software architecture, project structure and development rules.

---

## ProductBook.md

Describes every functional module of the application.

Explains how features should behave from the user's perspective.

---

## DesignBible.md

Defines the complete design system.

Includes colors, typography, spacing, reusable components and UI principles.

---

## MotionGuide.md

Defines animations, transitions and interaction principles.

Explains how the application should feel.

---

## SprintBook.md

Records the complete development history.

Tracks completed, current and future Sprints.

---

## Roadmap.md

Defines the long-term evolution of Praxis360.

# Philosophy

Praxis360 is developed as a long-term premium product.

Business vision always comes before technical implementation.

Every technical decision must improve the customer experience.

Consistency, simplicity and maintainability always take priority over development speed.
