# 1. Design Philosophy

Praxis360 delivers a premium, modern and reassuring user experience.

The interface should inspire confidence, clarity and simplicity.

The design language is inspired by:

- Apple
- Revolut
- Notion

Every screen should feel calm, elegant and easy to understand.

---

# 2. Brand Identity

Core values

- Trust
- Simplicity
- Elegance
- Professionalism

The interface should never feel overloaded.

Whitespace is an essential design element.

---

# 3. Color Palette

Primary

#01211E

Secondary

#4CB7A8

Accent

#0B1F3A

Neutral

#D4D2D6

Future versions may introduce semantic colors:

- Success
- Warning
- Error
- Information

---

# 4. Typography

Use a clean and modern sans-serif font.

Hierarchy

- Large Titles
- Section Titles
- Body Text
- Secondary Text
- Captions

Typography must remain consistent across the application.

---

# 5. Spacing System

Use consistent spacing throughout the application.

Recommended scale

4px

8px

12px

16px

24px

32px

48px

Never use arbitrary spacing.

---

# 6. Border Radius

Cards

Large rounded corners.

Buttons

Rounded but elegant.

Inputs

Soft corners.

The application should never feel sharp or aggressive.

---

# 7. Shadows

Use subtle elevation.

Avoid heavy shadows.

The interface should feel light.

---

# 8. Icons

Use a single icon family throughout the application.

Icons should remain simple and recognizable.

Avoid decorative icons.

---

# 9. Layout Principles

Each screen should have:

- One clear objective.
- Strong visual hierarchy.
- Generous spacing.
- Limited cognitive load.

Scrolling should feel natural.

---

# 10. Components

Reusable components include:

- P360Card
- DocumentCard
- CategoryCard
- SearchBar
- Navigation
- Buttons

Every reusable component belongs in Shared.

---

# 11. Forms

Forms should:

- Be simple.
- Minimize user effort.
- Clearly indicate required fields.
- Provide immediate validation feedback.

---

# 12. Navigation

Navigation should remain predictable.

Users should always know where they are.

Avoid unnecessary navigation depth.

---

# 13. Cards

Cards are a key element of Praxis360.

Every card should include:

- Clear title
- Optional subtitle
- Consistent spacing
- Soft shadow
- Rounded corners

Cards should remain visually lightweight.

---

# 14. Buttons

Buttons should clearly indicate priority.

Primary button

Main action.

Secondary button

Alternative action.

Danger button

Destructive action only.

---

# 15. Animations

Animations should always support usability.

Never animate simply for decoration.

Refer to MotionGuide.md for detailed animation specifications.

---

# 16. Accessibility

The application should provide:

- High contrast
- Readable typography
- Large touch targets
- Consistent navigation
- Accessible color usage

---

# 17. Responsive Design

The application is mobile-first.

Layouts should adapt gracefully to tablets.

Desktop support may evolve later.

---

# 18. Future Evolution

The Design System should continue growing while preserving consistency.

New components should follow the existing design language rather than introducing new visual styles.
