# 1. Motion Philosophy

Motion is not decoration.

Every animation must improve the user experience by providing context, feedback or guidance.

Animations should feel smooth, elegant and natural.

The application should never feel slow because of animations.

---

# 2. Animation Principles

Animations should always be:

- Smooth
- Subtle
- Fast
- Consistent
- Purposeful

Avoid unnecessary visual effects.

Never animate simply to impress.

---

# 3. Timing

Recommended durations:

- Quick feedback: 100–150 ms
- Standard transition: 200–300 ms
- Complex transition: 300–450 ms

Animations longer than 500 ms should remain exceptional.

---

# 4. Easing

Prefer natural acceleration and deceleration.

Avoid linear animations whenever possible.

Animations should feel organic.

---

# 5. Screen Transitions

Navigation between screens should feel fluid.

Transitions should reinforce the user's understanding of navigation.

Avoid abrupt screen changes.

---

# 6. Navigation

Navigation should provide immediate visual feedback.

Selected items should animate subtly.

Menus should open and close smoothly.

---

# 7. Cards

Cards may include:

- Gentle fade-in
- Slight elevation on hover (desktop)
- Soft press animation
- Smooth appearance

Animations should remain discreet.

---

# 8. Buttons

Buttons should provide instant feedback.

Examples:

- Press animation
- Ripple effect (if appropriate)
- Soft scaling
- Opacity change

Feedback should always be immediate.

---

# 9. Lists

List items should appear progressively.

Filtering and searching should animate smoothly.

Avoid sudden jumps in layout.

---

# 10. Loading States

Use lightweight loading indicators.

Prefer skeleton screens over large spinners when appropriate.

Loading animations should reassure users without distracting them.

---

# 11. Feedback

Important actions should provide clear visual confirmation.

Examples:

- Success
- Error
- Warning
- Information

Feedback should always be immediate.

---

# 12. Scanner

Scanner interactions should feel responsive.

Examples:

- Camera opening
- Capture feedback
- Processing indicator
- Successful scan confirmation

Animations should reinforce confidence.

---

# 13. Logo Animation

The Praxis360 logo is a distinctive part of the brand identity.

Signature animation:

- The circle of dots rotates smoothly around the logo.
- Movement is continuous and elegant.
- Speed remains slow and calming.
- The animation should never become distracting.

This animation may be used:

- Splash screen
- Loading screen
- Authentication
- Marketing materials

---

# 14. Accessibility

Animations must respect accessibility settings.

Where supported:

- Reduce motion should simplify or disable non-essential animations.
- Feedback animations should remain understandable without motion.

Accessibility always takes priority over visual effects.

---

# 15. Future Evolution

As Praxis360 evolves, new animations should extend the existing motion language rather than introducing unrelated styles.

Consistency is more important than complexity.
